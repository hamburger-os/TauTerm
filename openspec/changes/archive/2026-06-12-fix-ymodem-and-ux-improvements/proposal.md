## Why

用户在测试 TauTerm 串口终端时发现了 4 个影响核心体验的问题：YModem 文件发送误报"块0收到意外响应"导致传输失败、顶部快速连接栏缺乏实用功能按钮、会话关闭后无法记忆需每次重建、文件传输面板初始高度过小无法看清内容。这些问题直接影响终端日常使用的流畅度，需在下一轮迭代中集中修复。

## What Changes

- **YModem 发送修复**：修复单文件传输时 EOT 后的空块 0 被设备拒绝导致报错的问题——空块 0 失败时不再视为致命错误（文件数据已传输完成）
- **顶栏改为功能工具栏**：移除快速连接输入框，改为图标+文字的功能按钮组（新建会话、刷新端口、切换侧栏、文件传输、设置等），与新标签页按钮整合
- **会话持久化**：应用关闭时自动保存活跃会话配置到磁盘，启动时恢复标签页布局（不自动重连，保留连接参数供用户手动重连）
- **文件传输面板默认高度**：面板打开时默认高度从 24px 提升到 200px，确保首次打开即可看清传输历史与操作按钮

## Capabilities

### New Capabilities

- `toolbar-redesign`: 将顶部快速连接栏替换为功能图标工具栏，提供直观的常用操作入口
- `session-persistence`: 应用关闭时自动保存会话标签页配置，下次启动时恢复，提升多会话工作流的连续性

### Modified Capabilities

_无现有 spec 需修改。_

## Impact

- **前端组件**：`QuickConnectBar` → 重构为 `Toolbar`；`App.tsx` 移除快速连接相关逻辑，集成工具栏；`FileTransferPanel` 面板高度逻辑调整
- **Tauri 后端**：`commands.rs` 添加/调整会话持久化命令；`SessionManager` 已有保存/加载基础方法，需连接生命周期钩子
- **YModem**：`ymodem.rs` 的 `send()` 方法中，批次结束空块 0 的错误降级为警告而非致命错误
- **i18n**：工具栏按钮标签需新增翻译条目（`en-US.json`、`zh-CN.json`）
