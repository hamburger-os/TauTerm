## 新增需求

### Requirement: 多语言支持
系统必须支持多语言国际化，默认语言为简体中文（`zh-CN`），并允许用户切换至英文（`en-US`）。

#### Scenario: 应用默认语言
- **WHEN** 应用首次启动
- **THEN** 所有 UI 文本（按钮标签、状态消息、提示信息、菜单项）必须以简体中文显示

#### Scenario: 切换至英文
- **WHEN** 用户在设置中将语言切换为英文
- **THEN** 所有 UI 文本必须即时切换为英文，无需重启应用

#### Scenario: 切换回中文
- **WHEN** 用户在英文界面下将语言切换回中文
- **THEN** 所有 UI 文本必须即时恢复为简体中文

#### Scenario: 语言偏好持久化
- **WHEN** 用户切换语言后关闭并重新打开应用
- **THEN** 应用必须以用户上次选择的语言启动

### Requirement: 语言文件结构
语言资源必须以 JSON 格式存储，按功能模块划分命名空间，包含应用、串口、文件传输、设置和通用五个命名空间。

#### Scenario: 中文语言文件完整性
- **WHEN** 查看 `zh-CN.json`
- **THEN** 文件必须包含 `app`、`serial`、`transfer`、`settings` 和 `common` 五个命名空间，涵盖所有 UI 文本的翻译

#### Scenario: 英文语言文件完整性
- **WHEN** 查看 `en-US.json`
- **THEN** 文件必须包含与中文语言文件完全相同的键结构，值以英文填写

#### Scenario: 缺失翻译回退
- **WHEN** 某个文本键在当前语言文件中缺失
- **THEN** 系统必须回退到默认语言（中文）对应的文本

### Requirement: 语言切换 UI
设置界面必须提供语言切换控件，允许用户查看和更改当前语言。

#### Scenario: 显示当前语言
- **WHEN** 用户打开设置区域
- **THEN** 当前活跃语言（中文或 English）必须被明显标示

#### Scenario: 切换语言操作
- **WHEN** 用户点击语言切换按钮或在下拉菜单中选择另一语言
- **THEN** 语言切换必须立即生效，界面即时刷新为所选语言
