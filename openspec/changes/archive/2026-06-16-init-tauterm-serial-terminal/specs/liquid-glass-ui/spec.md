## 新增需求

### Requirement: Liquid Glass 设计语言
界面必须实现玻璃拟态设计系统，特征为磨砂玻璃面板、背景模糊、半透明表面和微细亮色边框。

#### Scenario: 玻璃面板渲染
- **WHEN** 应用窗口显示
- **THEN** 所有面板表面（侧边栏、工具栏、文件传输面板）必须以 `backdrop-filter: blur()`、半透明深色背景（`rgba` alpha < 0.15）和低透明度白色的 1px 微细边框渲染

#### Scenario: 深度和层次感
- **WHEN** 玻璃面板重叠（如文件传输面板滑过终端上方）
- **THEN** 重叠区域必须显示增强的模糊和变暗效果，透过各层营造视觉深度感

#### Scenario: 悬停和激活状态
- **WHEN** 用户悬停或交互 UI 元素（按钮、输入框、选择器）
- **THEN** 元素必须显示背景透明度、边框亮度的平滑过渡，以及微妙的发光效果

### Requirement: 暗色主题基础
界面必须在所有玻璃元素下使用带微妙渐变的深色背景作为基础层。

#### Scenario: 背景渐变
- **WHEN** 应用渲染
- **THEN** 根背景必须显示从深海蓝/深蓝到接近黑色的平滑渐变，在玻璃面板背后创造深度感

#### Scenario: 文字对比度
- **WHEN** 文字显示在玻璃面板上
- **THEN** 主文字必须使用高对比度亮色（`#e0e0e0` 或等效色），辅助文字必须使用柔和的灰色，确保在半透明背景上的可读性

### Requirement: 强调色和渐变
系统必须使用青绿到天蓝的强调渐变用于交互元素、焦点指示和品牌展现。

#### Scenario: 主按钮使用强调渐变
- **WHEN** 主操作按钮（连接、发送、接收）在默认状态下渲染
- **THEN** 必须以青绿到天蓝渐变作为其背景色

#### Scenario: 焦点指示
- **WHEN** 输入框获得焦点
- **THEN** 其边框必须以强调色动画变化，带有微妙的发光效果

#### Scenario: 进度指示
- **WHEN** 显示进度条（文件传输、连接状态）
- **THEN** 已填充部分必须使用强调渐变

### Requirement: 终端排版
终端视口必须使用优化的等宽字体以提高可读性，UI 装饰使用清爽的无衬线字体。两种字体均需支持中文字符渲染。

#### Scenario: 终端字体
- **WHEN** 终端渲染文字
- **THEN** 必须使用 JetBrains Mono（回退到 Cascadia Code、Fira Code 或系统等宽字体）以舒适的阅读大小显示

#### Scenario: UI 字体
- **WHEN** UI 装饰元素（标签、按钮、菜单）渲染文字
- **THEN** 必须使用 Inter（回退到系统无衬线字体），具有适当的字重和大小层次结构。中文字符必须能正常回退到系统默认中文字体

### Requirement: 可调整面板的响应式布局
应用布局必须适应窗口大小变化，并允许用户调整侧边栏和文件传输面板的大小。

#### Scenario: 窗口大小调整
- **WHEN** 用户调整应用窗口大小
- **THEN** 终端视口必须填满可用空间，侧边栏必须在其最小/最大约束内保持宽度，所有玻璃面板必须以新尺寸正确重新渲染模糊效果

#### Scenario: 可调整的侧边栏
- **WHEN** 用户拖动侧边栏与终端之间的分隔线
- **THEN** 侧边栏宽度必须在 200px（最小）和 400px（最大）之间平滑调整

#### Scenario: 可调整的文件传输面板
- **WHEN** 用户拖动文件传输面板与终端之间的分隔线
- **THEN** 面板高度必须在 150px（最小）和窗口高度的 50%（最大）之间平滑调整

### Requirement: 流畅动画和过渡
界面必须使用 CSS 动画和过渡处理状态变化、面板打开/关闭和交互反馈。

#### Scenario: 面板滑动动画
- **WHEN** 文件传输面板打开或关闭
- **THEN** 必须以流畅的滑动过渡（300ms ease-in-out）动画，无布局跳动或闪烁

#### Scenario: 连接状态过渡
- **WHEN** 连接状态变化（已断开 → 连接中 → 已连接 → 已断开）
- **THEN** 状态指示器必须以颜色过渡和微妙的脉冲效果动画变化

#### Scenario: 悬停微交互
- **WHEN** 用户悬停在按钮或交互元素上
- **THEN** 必须在 150ms 内发生微妙的缩放或亮度过渡
