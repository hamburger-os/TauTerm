# serial-terminal

## Purpose

定义串口终端功能要求，包括串口枚举、连接管理、数据收发和配置界面。v0.2 升级支持多会话并发和缓冲 I/O 通道。

## Requirements

### Requirement: 串口枚举
系统必须枚举主机上所有可用的串口，并返回其名称和描述信息。

#### Scenario: Windows 下列出可用端口
- **WHEN** 用户在 Windows 上打开端口选择下拉菜单
- **THEN** 系统必须显示所有可用的 COM 端口（如"COM1"、"COM3"），尽可能附带硬件描述

#### Scenario: Linux 下列出可用端口
- **WHEN** 用户在 Linux 上打开端口选择下拉菜单
- **THEN** 系统必须显示所有可用的 TTY 设备（如"/dev/ttyUSB0"、"/dev/ttyACM0"）

#### Scenario: 无可用端口
- **WHEN** 系统未连接任何串口设备
- **THEN** 端口列表必须显示空状态提示信息，表明未找到端口

#### Scenario: 刷新端口列表
- **WHEN** 用户点击端口选择器旁的刷新按钮
- **THEN** 系统必须重新扫描串口并更新列表

### Requirement: 串口连接
系统必须允许用户通过可配置的参数打开和关闭串口连接。

#### Scenario: 使用默认设置打开连接
- **WHEN** 用户选择端口并点击"连接"，使用默认设置（115200-8-N-1，无流控）
- **THEN** 端口必须成功打开，终端必须显示"已连接到 <端口名>"

#### Scenario: 使用自定义设置打开连接
- **WHEN** 用户配置波特率 921600、8 数据位、偶校验、2 停止位和 RTS/CTS 流控，然后点击"连接"
- **THEN** 端口必须以指定的配置打开

#### Scenario: 端口已被占用
- **WHEN** 用户尝试连接到已被其他应用打开的端口
- **THEN** 系统必须显示错误信息，表明端口不可用

#### Scenario: 断开端口连接
- **WHEN** 用户在活跃连接上点击"断开"
- **THEN** 端口必须被关闭，终端必须显示"已断开"

#### Scenario: 会话期间端口消失
- **WHEN** 已连接的 USB 串口适配器被物理拔出
- **THEN** 系统必须检测到断开、关闭端口句柄，并显示"设备已断开"通知

### Requirement: 串口数据接收与显示
系统必须从打开的串口读取数据，并实时在终端仿真器中显示。接收循环必须与写入操作公平交替。系统必须支持三种数据模式：Text（纯文本）、HEX（十六进制）、Dual（双模）。

#### Scenario: 显示接收到的文本数据
- **WHEN** 串口收到文本数据，且数据模式为 Text
- **THEN** 数据必须在 xterm.js 终端视口中以正确的字符渲染显示

#### Scenario: 处理 ANSI 转义序列
- **WHEN** 接收到 ANSI 转义序列（颜色、光标移动、清屏）
- **THEN** 终端必须正确渲染（彩色文字、光标定位、屏幕清除）

#### Scenario: 处理二进制数据
- **WHEN** 接收到非有效 UTF-8 的原始二进制数据
- **THEN** 终端必须显示替换字符或十六进制表示，且不崩溃

#### Scenario: 同时收发数据
- **WHEN** 串口同时接收数据和用户输入命令
- **THEN** 接收和发送必须交替进行，任何一方不被另一方阻塞超过 100ms

#### Scenario: HEX 十六进制模式
- **WHEN** 数据模式设置为 HEX
- **THEN** 接收到的数据必须以固定 16 字节一行的 Hex Dump 格式显示，包含偏移量、十六进制值和 ASCII 预览列

#### Scenario: Dual 双模显示 — 双栏布局
- **WHEN** 数据模式设置为 Dual
- **THEN** 显示区域必须分为左右两栏：左侧显示 ASCII 文本，右侧显示 HEX 十六进制，两栏之间可通过拖拽分隔条调整宽度，拖拽时分隔条必须有视觉高亮反馈，左右滚动同步

#### Scenario: Dual 双模显示 — 时间戳
- **WHEN** Dual 模式下收到数据帧
- **THEN** 每帧数据前必须附加毫秒精度时间戳 `[HH:MM:SS.mmm]`，仅左侧 ASCII 栏显示时间戳，右侧 HEX 栏通过行对齐与左侧保持对应

#### Scenario: Dual 双模显示 — 分帧
- **WHEN** Dual 模式下收到数据
- **THEN** 系统必须按 `\r\n`、`\n` 或 `\r` 分隔符自动分帧，超时（默认 50ms，可在连接配置中设置，范围 5–500ms）未收到新数据时剩余缓冲数据作为最后一帧输出

#### Scenario: Dual 双模显示 — 收发区分
- **WHEN** Dual 模式下发送数据
- **THEN** 发送的数据（TX）必须以区别于接收数据（RX）的颜色高亮显示（TX 青色，RX 默认色）

#### Scenario: Dual 双模显示 — 方向标签
- **WHEN** Dual 模式下显示行数据
- **THEN** 每行必须附加 `[RX]` 或 `[TX]` 方向标签，标签颜色与数据方向颜色一致，时间戳以灰色显示

#### Scenario: Dual 双模显示 — 帧分隔符处理
- **WHEN** 按 `\r\n`、`\n` 或 `\r` 分隔符分帧后
- **THEN** 帧数据尾部的 `\r` 和 `\n` 字节必须被去除后再显示（分隔符由行结构体现），帧中间的控制字符 `\r` 渲染为 `␍`、`\n` 渲染为 `␊`

#### Scenario: Dual 双模显示 — 数据到达时保持滚动位置
- **WHEN** Dual 模式下用户手动滚离底部且新数据到达
- **THEN** 新数据行必须被追加但不应改变当前滚动位置；若用户在底部则自动跟踪最新数据

#### Scenario: Dual 双模显示 — 行缓冲上限
- **WHEN** Dual 模式行数达到配置的行缓冲上限（默认 10,000 行，范围 1,000–100,000）
- **THEN** 系统必须移除最早的行以保持总行数不超过上限

### Requirement: 串口数据发送
系统必须允许用户在终端中输入并将按键发送到串口。I/O 通道必须使用带缓冲的 `sync_channel(32)` 替代无缓冲 rendezvous 通道。I/O 循环必须采用公平读写调度，读操作不再跳过写检查。

#### Scenario: 发送键入字符
- **WHEN** 用户在连接串口时输入字符
- **THEN** 每个字符必须实时通过串口发送，即使串口正在持续接收数据

#### Scenario: 发送特殊按键
- **WHEN** 用户按下 Enter、Tab、Escape 或方向键
- **THEN** 对应的控制序列必须通过串口发送，Enter 键必须在 100ms 内被写入设备

#### Scenario: 快速连续输入
- **WHEN** 用户在连接状态下快速连续输入 20 个字符
- **THEN** 所有字符必须被缓冲并通过串口发送，不阻塞 UI 线程

#### Scenario: 粘贴文本
- **WHEN** 用户在连接状态下从剪贴板粘贴多行文本
- **THEN** 粘贴的文本必须通过串口发送

### Requirement: 串口配置界面
系统必须提供配置所有标准串口参数的 UI 控件。

#### Scenario: 配置波特率
- **WHEN** 用户从下拉菜单选择波特率
- **THEN** 可选选项必须包括：110、300、600、1200、2400、4800、9600、14400、19200、38400、57600、115200、230400、460800、921600，以及一个自定义输入框

#### Scenario: 配置数据位
- **WHEN** 用户从下拉菜单选择数据位
- **THEN** 可选选项必须为 5、6、7 和 8

#### Scenario: 配置校验位
- **WHEN** 用户从下拉菜单选择校验位
- **THEN** 可选选项必须为：无、偶校验、奇校验、标记、空格

#### Scenario: 配置停止位
- **WHEN** 用户从下拉菜单选择停止位
- **THEN** 可选选项必须为 1、1.5 和 2

#### Scenario: 配置流控
- **WHEN** 用户从下拉菜单选择流控
- **THEN** 可选选项必须为：无、RTS/CTS（硬件）、XON/XOFF（软件）

### Requirement: 多会话串口支持
系统必须支持同时打开多个串口会话，每个会话拥有独立的 I/O 线程和通道。

#### Scenario: 同时连接两个串口
- **WHEN** 用户在标签页 A 连接 COM3，在标签页 B 连接 COM5
- **THEN** 两个串口同时独立工作，互不干扰

#### Scenario: 一个会话断开不影响其他
- **WHEN** COM3 设备被物理拔出
- **THEN** COM3 的标签页显示断开状态，COM5 的标签页继续正常工作
